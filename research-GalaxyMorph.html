In order to unveil the true relation between the structures and evolutionary history of galaxies, one requires an "accurate" classification of galaxy's morphologies. This can be intepreted in two perspectives: (1) correctly classifying the morphology of a galaxy based on current visual classification schemes such as Hubble sequence (Section I); or (2) creating a new categorisation scheme that are physically meaningful for galaxy morphology (Section II).

Section I: Fast classification with supervised machine learning

Visual classification is still one of the popular approaches to classify the morphology of galaxies, despite that two experts may find their classifications inconsistent with each other (Kormendy 1982 and Abraham 1999). However, visual classification is a very laborious process; even with the success of citzen science such as Galaxy Zoo series projects, it could still takes couple years to have the classification of a million of galaxies. Supervised machine learning (ML) is therefore a great tool to improve and accelerate the classification process.

Optimal ML techniques for galaxy morphological classification

In Cheng 2020a, we investigated several common supervised ML techniques [including Convolutional Neural Network (CNN), K-nearest neighbour, logistic regression, Support Vector Machine, Random Forest, and Neural Networks] on binary classification of galaxy morphology (i.e. spiral and elliptical galaxy) using the imaging data from the Dark Energy Survey (DES) Year 1 data release. We tested not only different techniques but also different types of inputs such as (1) balanced and imbalanced number of data between the target classes and (2) with different scales such as linear, logrithmic, and gradient in pixel values. In Fig.1, we investigate the impact of having imbalanced number of data between the two target types. The dataset 1 (yellow) and 3 (cyan) have 3 times more spiral galaxies than elliptical galaxies, while the two types have the same number of data in the dataset 2 (red) and 4 (navy blue). The metric used here is called "recall" which represents the number of data that is correctly classified within their true classes. Regardless of the methodology, the imbalanced datasets present an biased result towards the majority -- machines tend to have better performance in classifying spiral galaxies than elliptical galaxies, while the balanced datasets show a fair performance between two types.

The comparison of the accuracy between different methods and different types of input data is shown in Fig.2. The accuracy is defined as the number of galaxies that are correctly classified within all galaxies. Fig.2 shows that the optimal methodology for this task is to apply CNNs on balanced datasets with the combination of linear and gradient images. This gives an initial accuracy of ~0.95 to classify galaxies into spiral and elliptical galaxies.
